Overview


A producer is the one which publishes or writes data to the topics within different partitions.

Producers automatically know that, what data should be written to which partition and broker.



How does the producer write data to the cluster


  • 1. Message keys
  • 2. Acknowledgment


1. Message keys

If the producers apply key over the data(key != null), that data will always be sent to the same partition.

If the producers does not apply the key while writing the data(key = null), it will be sent in a round-robin manner. This process is called load balancing.



2. Acknowledgment

acks = 0, means producer sends the data but does not wait for the acknowledgment(data loss).

acks = 1, means producer will wait for leader's acknowledgment(limited data loss).

acks = all means acknowledgment is done by both leader and it's followers(no data loss).
