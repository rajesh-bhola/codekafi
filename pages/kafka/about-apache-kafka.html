Apache Kafka with Advantages & Disadvantages

Rajesh Bhola
Jr. Software Engineer at Girmiti
Exp: 2.5+ years

Published on : 14-July-2021

What is Apache Kafka


Apache kafka is a distributed publish-subscribe messaging system and a roubust queue that can handle a high volume of data and enables you to pass messages from one end-point to another.

Kafka is suitable for both online and offline message consumption.

Kafka, which exchanging of data between applications, servers and processors as well.



Originally developed by LinkedIn and donated to Apache Software Foundation. Currently it is maintained by Confluent.

Kafka messages are persisted on the disk and replicated within the cluster to prevent data loss.

Kafka is built on top of zookeeper service. It integrates very well with Apache Storm and Spark for real-time streaming data analysis.



Why Apache Kafka


It capables of handling millions of data or messages per second.

It works as a mediator between source system and target system. Thus, source system(Producer) data is sent to Apache Kafka, where it decouples the data and the target system(Consumer) consumes the data from Kafka.

The offsets for a partition are infinite.

It is having extremely high performance i.e it has low latency value less than 10ms.(message delivery)

Organizations such as LinkedIn, NetFlix, Twitter, Uber, Walmart, Oracle, Mozilla and over thousands of IT firms uses Apache Kafka.

It give guarantee for fault tolerance in the presence of machine failures.

Examples:

Consumer successfully consumes the message that was delivered by the producer. But, the consumer fails to process the message back due to backend database failure or due to presence of a bug in the consumer code. In such situation consumer is unable to consume the message again. Apache kafka has resolved the problem by reprocessing the data.





It has the ability to handle a large number of diverse consumers.

Kafka is very fast and performs 2 million writes per sec.

Kafka persists all data to the disk, which essentially means that all the writes go to the page cache of the OS(RAM). This makes it very efficient to transfer data from page cache to a network socket.



Benefits


Reliability - Kafka is distributed, partitioned, replicated and fault tolerance.

Scalability - Kafka messaging system scales easily without downtime.

Durability - Kafka uses "distributed commit log" which means messages persists on disk as fast as possible, hence it is durable.

Performance - It maintains stable performance many terra bytes of messages are stored.(high throughput & low latency)



Use Cases


Messaging Broker - Used for to decouple processing from data producers.
# better throughput, built-in partitioning, replication and fault tolerance.

Website activity tracking - site activity like pageview, searches, clicks, other actions user may take etc.

Metrics - Used for operational monitoring data, this involves aggregating statistics from distributed applications.

Log Aggregation - Collects physical log files of server and puts them in a central place for processing. This allows lower-latency processing.

Stream Processing - Read data from a topic, process it, write that processed data over a new topic. This new topic containing the processed data available to users and apllications.

Event Sourcing - Handling large amounts of logs data make it an excellent backend for building an application.

Commit Logs - Helps replicate data between nodes and act as a re-syncing mechanism failed nodes to restore their data.



Advantages


Low Latency - because it decouples the message upto 10ms.

High Throughput - handles high volume and high velocity.

Durability - replication feature, which makes data to persist more on the cluster over a disk.

Easily Accessible - data get stored in kafka. It becomes easily accessible to anyone.

Distributed System - contains distributed architecture which makes it scalable(partitioning & replication).

Real-time Handling - handles real-time data pipelines.

Low latency - because it decouples the message upto 10ms.

Batch Processing - processing the messages in a bacth.



Disadvantages


Message Tweaking Issues - it works well if the message does not need to change.

Reduces Performance - brokers and consumers reduces performance by compressing and decompressing the data flow.

Clumpsy Behaviour - bit clumpsy when the number of queues increases in the clustger.
