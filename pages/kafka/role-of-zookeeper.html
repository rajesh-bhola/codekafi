Roles of Zookeeper


A zookeeper is used to store information about the kafka cluster and details of the consumer clients.

It manages brokers by maintaining a list of them.

A zookeeper is responsible for choosing a leader for the partitions.



If any changes like a broker die, new topics, etc. occurs, the zookeeper sends notifications to kafka.

Each partition contains messages in an immutable ordered sequence.

A zookeeper is designed to operate with an odd number of kafka servers.



Zookeeper has a leader server that handles all the writes and rest of the servers are followers who handles all the reads.

A user doesn't directly interact with zookeeper but via brokers.

The offsets for a partition are infinite.

No kafka server can run without a zookeeper.
