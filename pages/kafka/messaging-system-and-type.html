Overview


Regarding data, we have two main challenges.

  • 1. How to collect large volume of data
  • 2. How to analyze the collected data

To overcome those challenges, me must need a messaging system.



What is messaging system


A messaging system is responsible for transferring data from one application to another application. So the applications can focus on data.



What is distributed messaging system


Distributed system with multiple servers, each of which is responsible for holding data and performing operations on that data.

Distributed messaging is based on the concept of message queueing.

Messages are queued ashynchonously between client applications and messaging system.



Types of messaging patterns


  • 1. Point-to-Point
  • 2. Publish-Subscribe

Notes: Most of the messaging patterns follow pub-sub.



1. Point-to-Point

In a point-to-point messaging system, the sender delivers the message to the queue.

One or more consumers can consume the messages in the queue, but a particular message can be consumed by a maximum of one consumer only.

Once a consumer reads a message in the queue, it disappears from that queue.



2. Publish-Subscribe

In a publish-subscribe messaging system, the sender delivers the message to the topics.

Consumers can subscribe to one or more topic and consume all the messages from that topic.

In pub-sub system, message producers are called Publishers and message consumers are called Suscribers.


Examples:

Dish TV, which publishes different channels like sports, movies, music etc. and anyone can subscribe to their own channels and get them whenever their subscribed channels are available.
