Overview





KsqlDB is an event streaming database to help developers to create stream processing applications.

Process streams of unbounded kafka data in real-time with KSQL.



Components of ksqlDB


Collections

- Provides durable storage for sequences of events.
- KsqlDB offers two kinds of collections Streams and Tables.


Materialized views

- Materialized views are derived representations of streams or tables.
- They enable you to create new collections over existing streams or tables.
- Materialized views are especially useful for maintaing aggregated tables of data.


Queries

- Queries enable you to access these materialization from applications or microservices.
- Applications can query materialized views with Pull Queries and Push Queries



What can we do with ksqlDB


Data Exploration


- KSQL is an easy way to explore kafka topic data.

- Check the list of topics available in the kafka cluster.

ksql > show  topics;

- Explore data in the kafka topic.

ksql > print  "kafka_topic"  from  beginning;


Streaming ETL


- Apache kafka is popular choice for powering data pipelines.
- ksqlDB makes it simple to transform data within the pipeline, reading messages to clearly in another system.



Anomaly Detection


Identify patterns and spot anomalies in real-time data with millisecond latency and to handle fraudulent activities.



Real-time Monitoring & Analytics


Track, understand & manage infrastructure, applications, data feeds by quickly building real-time dashboards, generating metrics and creating custom alerts and messages.
